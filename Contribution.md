---

Development Workflow

1. Each team member will work on assigned tasks.
2. Changes will be committed regularly with meaningful commit messages.
3. Code will be pushed to the respective branch.
4. Pull Requests will be created before merging changes.
5. Team members will review code when required.
6. Approved changes will be merged into the main branch.

---

Commit Guidelines

- Every team member must contribute through their own GitHub account.
- Commits should be made regularly throughout development.
- Commit messages should clearly describe the changes made.
- Pull Requests should be used before merging major changes.

Sample Commit Messages

- feat: create event card component
- feat: add authentication API
- feat: integrate MongoDB database
- fix: resolve registration validation issue
- docs: update project documentation
